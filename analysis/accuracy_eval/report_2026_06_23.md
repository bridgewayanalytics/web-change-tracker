# NAIC Alert Agent — Accuracy Evaluation Report
**Date:** June 23, 2026  
**Sample:** 20 alerts, stratified across 12 alert types  
**Method:** Field-by-field comparison against live NAIC pages + agent output schema  
**Evaluator:** In-session analysis with full project context  
**Scoring:** 2 = Correct · 1 = Partial / minor issue · 0 = Incorrect / missing · N/A = not applicable to this alert type

---

## Executive Summary

The alert agent performs **strongly on core fields** — correctly identifying what changed, where, and which organization is involved in roughly 90% of cases. URL and filename capture is near-perfect. The system has two distinct failure modes: (1) **two legacy-schema rows** (rows 11 and 20) where field types break entirely (null instead of string, string instead of dict), pulling the overall score down; and (2) **event title formatting** is inconsistent across rows and the agent occasionally misclassifies an article-about-a-meeting as a calendar event.

Excluding the two schema-version outliers, the remaining 18 rows score **91.7% overall**. Including them: **85.4%**.

The weakest areas are event title formatting (61.5%), chronicle topic assignment (70%), and library item title quality (73.3%). These are addressable through prompt tuning. The strongest areas are URL/filename capture (100%), alert type (92.5%), and core text fields (92.5%).

## Key Issues Identified

### 1. Schema Version Mismatch — Rows 11 and 20 (Critical)

These rows were generated before the flat schema was fully standardized. Fields that should be strings contain `null`, fields that should be dicts (`{status, title}`) are plain strings, and `organization` is a plain string instead of a list. This causes:

- `event_start_date_time`, `event_end_date_time`, `event_call_in_number_access_code` → `null` (should be `"N/A"`)
- `library_item_preliminary_title` → `"APF 2025-18"` (should be `{"title": ..., "status": ...}`)
- `agenda_item_title_chronicle_topics` → `null` (should be `[...]`)
- `is_the_alert_relevant_for_an_art_newsreel_article` → `null`

**Impact:** These rows cannot be used by the dashboard or Bubble sync. They represent a bug that was fixed but the old rows were never backfilled.  
**Fix:** Backfill rows 11 and 20 through the rerun pipeline.

---

### 2. Newsroom Article Misclassified as New Meeting — Row 16 (High)

A Newsroom page listing a "National Meeting News" article about the upcoming NAIC Spring National Meeting was classified as `alert_type: "New Meeting"`. The agent extracted the Spring National Meeting as a calendar event (`event_title: "NAIC Spring National Meeting"`) even though the source was a news article, not an event posting. The page had no event date, no Webex link.

**Correct type:** `"Other"` or `"New or Updated Report or Other Resource"`  
**Root cause:** The agent conflated the content of a news article (which mentioned a meeting) with an actual new meeting posting.  
**Impact:** This generates a spurious meeting event record in Bubble with no dates.

---

### 3. Event Title Formatting Inconsistency (Medium)

Three distinct formats appear across rows, none defined in the system prompt:

| Format | Example | Rows |
|--------|---------|------|
| `"NAIC Interim Meeting: [Group]"` | `"NAIC Interim Meeting: Life Risk-Based Capital (E) Working Group"` | 8, 10 |
| `"[Group] - [MMDDYYYY]"` | `"Life Actuarial (A) Task Force - 06252026"` | 7, 9, 12, 19 |
| `"[Full description]"` | `"Joint Meeting of the Property and Casualty Risk-Based Capital (E) Working Group..."` | 5 |
| `"[Short name]"` | `"Reinsurance (E) Task Force Meeting"` | 17 |

This inconsistency makes event-level deduplication in Bubble unreliable (match_key uses event_title for lookup).  
**Fix:** Define an explicit event_title format convention in the system prompt. Recommended: `"[Group Full Name] — [Month DD, YYYY]"`.

---

### 4. Chronicle Topics Frequently N/A (Medium)

The `chronicle_topics` array inside `agenda_item_title_chronicle_topics` is `["N/A"]` in 10 of 20 rows. The field should map to the NAIC Chronicle taxonomy (87 topics). When populated correctly (rows 6, 8, 10, 15, 17, 18), it's accurate. The agent appears to default to N/A when uncertain rather than making a best-effort classification.

**Examples of missed opportunities:**
- Row 1 (SHIP/MHP letter): could tag "Health Insurance", "State Regulation"
- Row 3 (Health RBC data analysis): could tag "Risk-Based Capital"
- Row 12 (Private Credit presentation): could tag "Capital Markets", "Private Equity"
- Row 9 (AG 55): could tag "Actuarial Guidelines"

**Fix:** Add the full Chronicle topic taxonomy list to the agent context with examples of when to assign each.

---

### 5. Library Item Title Quality — Filename vs. Descriptive Title (Medium)

In 3 rows, the agent uses the raw filename as the document title instead of a human-readable name:

- Row 6: `"GOES Model Change Templates 052826_0.xlsx"` (filename)
- Row 15: `"APF 2025-14"` (just the proposal number, no description)
- Row 20: `"APF 2025-18"` (same issue)

The library_item_preliminary_title is used as the Bubble record Name — it should be descriptive and readable.  
**Fix:** Add explicit instruction: "Derive the preliminary title from the document name as it appears on the page or in surrounding text, not from the URL or filename alone."

---

### 6. Description Brevity for RFC Rows (Low)

Rows 6 and 18 (GOES exposure drafts) share an identical, sparse description:  
`"New library item posted as part of the 2026 GOES exposure materials for comment."`

This adds little value. The description should explain what the document contains and what stakeholders are being asked to comment on.

---

### 7. Newsreel Relevance — Borderline Decisions (Low)

Two cases where "Yes" is questionable:
- **Row 4**: New meeting scheduled for HRBC WG. Meeting announcements are generally not newsreel articles unless the agenda has significant content. Score: borderline correct.
- **Row 19**: June 18 meeting extended by 30 minutes. A duration change alone is not newsreel-relevant. Score: incorrect "Yes."
- **Row 12**: Financial Stability TF agenda for S&P Private Credit presentation. "Yes" with `details: "N/A"` — the details should explain the newsreel rationale.
